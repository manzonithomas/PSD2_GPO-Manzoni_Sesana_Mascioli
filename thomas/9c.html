L'Era dell'
Open Banking.

L'entrata in vigore della PSD2 ha segnato una transizione epocale, spostando il baricentro dal controllo centralizzato delle banche verso un ecosistema aperto e interoperabile. Non è solo una legge: è una rivoluzione infrastrutturale.

Il cuore della trasformazione risiede nell'apertura delle API bancarie a terze parti autorizzate e nell'obbligo della Strong Customer Authentication, ridefinendo sicurezza e flussi transazionali a livello europeo.

Art. 4 RTS — EBA

Strong Customer Authentication

La SCA impone che ogni accesso al conto o inizializzazione di pagamento sia validato con almeno 2 fattori indipendenti. La compromissione di un elemento non deve facilitare la violazione del secondo — difesa a più livelli.

ConoscenzaQualcosa che solo tu SAI — Password, PIN
PossessoQualcosa che solo tu HAI — Smartphone, Token hardware, SIM
InerenzaQualcosa che solo tu SEI — Impronta digitale, Face ID, biometria vocale
0 / 2 fattori richiesti
Evoluzione: dai vecchi OTP via SMS alle notifiche push firmate digitalmente all'interno di app bancarie sicure — molto più resistenti all'intercettazione.

3-D Secure: 1.0 vs 2.0

La v2.0 supera i limiti della v1.0 — scarsa compatibilità mobile, password statiche, esperienza frammentata — grazie a SDK nativa, ML e oltre 150 punti dati per la valutazione del rischio.

Parametro 3DS 1.0 3DS 2.0 ✓
Integrazione Browser (iframe / redirect) SDK nativa mobile + browser
Dati scambiati ~15 elementi 150+ elementi
Autenticazione Password statica / OTP SMS Biometria + notifica push firmata
User Experience Friction (interruzioni frequenti) Frictionless / Challenge selettivo
Analisi del rischio Statica e limitata Machine Learning in tempo reale (RBA)
Risk-Based Authentication (RBA): Se il rischio è basso (dispositivo noto, comportamento coerente), la transazione passa in modalità frictionless — nessuna interruzione. In caso di anomalie scatta il Challenge Flow con SCA obbligatoria.

Durata dei Token OAuth 2.0

Il meccanismo dei token separa l'autenticazione dell'utente dall'autorizzazione dell'app, senza mai condividere username e password della banca.

Authorization Code
< 10 minuti
Codice temporaneo scambiato subito con la TPP. Usato una sola volta per ottenere i token finali.
Access Token
30 – 60 min
Autentica ogni singola chiamata API. Scade rapidamente per limitare i danni in caso di intercettazione.
Refresh Token
Giorni / Mesi
Rinnova l'Access Token senza coinvolgere l'utente. SCA obbligatoria ogni 90–180 giorni per gli AISP.

Come funziona il SIM Swap

Il criminale ottiene un duplicato della tua SIM fingendosi te all'operatore. Da quel momento intercetta tutti gli SMS, inclusi gli OTP — rendendo inutile l'SMS come secondo fattore SCA.

Vittima
Fase 1 → Il criminale raccoglie i dati personali della vittima tramite phishing o data breach precedenti.
Fase 2 → Si reca dall'operatore con documenti falsi: "Ho perso la SIM, ho bisogno di un duplicato."
Fase 3 → L'operatore attiva la nuova SIM. La SIM originale si disattiva — la vittima nota solo assenza di segnale.
Fase 4 → La banca invia l'OTP. Il criminale riceve l'SMS sul suo telefono e svuota il conto con bonifici istantanei (SCT Inst).
🛡️ Difese: PIN SIM abilitato, app authenticator (TOTP) invece degli SMS, alert di login. La PSD3 prevede l'abbandono definitivo degli SMS come fattore SCA.
